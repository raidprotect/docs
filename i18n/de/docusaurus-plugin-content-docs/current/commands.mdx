Alle RaidProtect-Befehle, gruppiert nach Kategorie. Klicken Sie auf den Namen eines Befehls, um zu seiner detaillierten Dokumentation zu gelangen.

## Legende der Prototypen {#legend}

- `(arg)`: **erforderliches** Argument.
- `[arg]`: **optionales** Argument.
- `[--flag]`: optionales boolesches Flag (Slash: `flag: true`, Nachricht: `--flag`).
- `[--flag=wert]`: optionales Flag mit Wert.
- `(@mitglied)` / `(#kanal)` / `(@&rolle)`: Präfix, das den erwarteten Typ angibt.

## ⚙️ Zwei Möglichkeiten, einen Befehl zu verwenden {#usage}

Beide Formate bieten **genau die gleichen Möglichkeiten**: Alles, was per Slash machbar ist, ist auch per Nachricht machbar — und umgekehrt. Der Unterschied liegt nur in der Art und Weise, wie der Befehl eingegeben wird.

### Slash-Befehle: am einfachsten {#slash}

Tippen Sie `/` in einen Kanal, wählen Sie den Befehl, und **Discord schlägt Ihnen jeden Parameter über eine spezielle Oberfläche vor** (Benutzer-Auswahl, Kanal-Auswahl, Autocomplete für Dauern usw.). Das ist die standardmäßig empfohlene Methode — Sie müssen sich nichts merken.

```
/ban @mitglied grund: scam-spam
```

### Nachrichtenbefehle: am komplexesten {#prefix}

Sobald das [Präfix in `/settings` aktiviert ist](./guides/prefix.md), können Sie die Befehle auch als klassische Nachricht schreiben (`?ban …`, `?kick …` usw.). Keine grafische Oberfläche: Sie müssen **die Syntax kennen** (positionelle Argumente, benannte Flags, freies Argument am Zeilenende). Im Gegenzug ist das schneller über die Tastatur für Moderatoren, die die Befehle beherrschen.

#### Positionelle Argumente

In der Reihenfolge des Prototyps angegeben, ohne Namen. Das sind die Hauptargumente des Befehls.

```
?ban 123456789012345678
?slowmode 10s
```

#### Benannte Flags (`--name=wert`)

Für sekundäre Optionen. Die Flags können **überall** im Befehl platziert werden, in beliebiger Reihenfolge.

- `--silent` (boolesch): aktiviert die Option einfach.
- `--delete-messages=7d`: gibt der Option einen Wert.

```
?ban @mitglied --delete-messages=7d --silent grund des bans
?tempban @mitglied 1d --silent er hat es wieder getan
```

#### „Rest"-Argument (freier Text)

Das letzte Argument bestimmter Befehle (typischerweise `[grund]`) erfasst **alles, was Sie danach schreiben**, einschließlich Leerzeichen, ohne Anführungszeichen setzen zu müssen.

```
?warn @mitglied du hast wieder Regel 3 des Servers verletzt
```
