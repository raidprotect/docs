El historial de sanciones de RaidProtect te permite rastrear y gestionar todas las sanciones aplicadas en tu servidor. Este sistema centraliza todas las acciones de moderacion en una base de datos consultable y editable, facilitando el trabajo de tu equipo de moderacion.

## ‚ùì Como funciona el historial {#working}

El historial de sanciones registra automaticamente todas las acciones de moderacion realizadas en tu servidor:

- **Sanciones manuales**: Todos los comandos de moderacion (`/ban`, `/tempban`, `/kick`, `/timeout`, `/warn`) se registran automaticamente en el historial.
- **Sanciones automaticas**: Las sanciones aplicadas por el anti-spam tambien se agregan al historial de sanciones.
- **Baneos, expulsiones y tiempos fuera**: Las sanciones aplicadas sin usar RaidProtect se agregan al historial.
- **Automod de Discord**: Las sanciones aplicadas por el automod de Discord tambien se agregan.
- **Notificaciones de sancion**: Los miembros sancionados reciben un mensaje privado informandoles sobre la sancion y su razon. El bot tambien envia un acuse de recibo confirmando el envio de esta notificacion.

:::info
Todas las sanciones registradas contienen: el usuario sancionado, el moderador responsable, la razon (hasta 512 caracteres), la fecha y hora, asi como el tipo de sancion y si el usuario fue notificado.
:::

## üîç Buscar sanciones {#search}

El comando `/sanctions search` te permite buscar sanciones en el historial segun diferentes criterios.

Usa el comando: ```/sanctions search [user] [moderator] [type] [date] [status]```

- `[user]`: Buscar todas las sanciones de un usuario especifico.
- `[moderator]`: Buscar todas las sanciones aplicadas por un moderador especifico.
- `[type]`: Filtrar por tipo de sancion (Ban, Softban, Unban, Kick, Timeout, Untimeout, Warn).
- `[date]`: Filtrar por fecha de sancion.
- `[status]`: Filtrar por estado de sancion (Activa, Expirada, Cancelada).

![Captura de pantalla de la busqueda de sanciones](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-sanctions-search.webp)

:::tip
Puedes combinar multiples criterios para refinar tu busqueda. Por ejemplo, buscar todos los baneos realizados por un moderador especifico.
:::

## ‚ÑπÔ∏è Ver una sancion {#info}

El comando `/sanctions info` te permite obtener informacion detallada sobre una sancion especifica.

Usa el comando: ```/sanctions info (id)```

Reemplaza `(id)` con el identificador de la sancion que deseas ver.

![Captura de pantalla de la informacion de sancion](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-sanctions-info.webp)

## ‚úèÔ∏è Editar una sancion {#edit}

El comando `/sanctions edit` te permite modificar la razon de una sancion existente, util para corregir un error o agregar detalles.

Usa el comando: ```/sanctions edit (id) (new_reason)```

Reemplaza `(id)` con el identificador de la sancion a modificar y `(new_reason)` con la nueva razon (maximo 512 caracteres).

![Captura de pantalla de la edicion de sancion](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-sanctions-edit.webp)

:::warning
Editar una sancion actualiza el registro en el historial pero no cambia la sancion aplicada en Discord (por ejemplo, un usuario baneado seguira baneado).
:::

## üóëÔ∏è Eliminar una sancion {#delete}

El comando `/sanctions delete` te permite eliminar una sancion del historial. Esta accion es irreversible.

Usa el comando: ```/sanctions delete (id)```

Reemplaza `(id)` con el identificador de la sancion a eliminar.

![Captura de pantalla de la eliminacion de sancion](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-sanctions-delete.webp)

## Estado de las sanciones {#status}

Las sanciones pueden tener diferentes estados:

| **Estado**    | **Emojis**                                                                                                                  | **Significado**                                           |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------| --------------------------------------------------------- |
| `Active`      |                                                                                                                             | La sancion esta en curso.                                 |
| `Expired`     | <Icon src="/img/icons/SanctionStatusEXPIRED.svg" alt="icon SanctionStatusEXPIRED" title=":iconSanctionStatusEXPIRED:"/>     | La sancion ha expirado.                                   |
| `Canceled`    | <Icon src="/img/icons/SanctionStatusCANCELED.svg" alt="icon SanctionStatusCANCELED" title=":iconSanctionStatusCANCELED:"/>  | La sancion ha sido cancelada por un moderador.            |
| `Failed`      | <Icon src="/img/icons/SanctionStatusFAILED.svg" alt="icon SanctionStatusFAILED" title=":iconSanctionStatusFAILED:"/>        | La sancion ha fallado (permisos insuficientes).           |

### Manejo de duplicados {#duplicates}

Cuando se encuentra una sancion activa y anterior, puedes:

- **Fusionar** las dos sanciones: las duraciones se sumaran si es posible y las razones se concatenaran.
- **Cancelar** la sancion activa y crear una nueva sancion.
- **Sobrescribir** la sancion activa con la informacion de la sancion ingresada.

## üìä Registros de sanciones {#logs}

Para una mejor organizacion, puedes configurar un canal de registros dedicado especificamente a sanciones, separado de tus registros generales.

![Captura de pantalla de los registros de sanciones](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-sanctions-logs.webp)

### Configurar el canal de registros de sanciones {#config-logs}

![Captura de pantalla de la configuracion de registros de sanciones](../../../../en/docusaurus-plugin-content-docs/current/assets/rp-settings-sanctions-logs.webp)

1. Usa el [comando `/settings`](../setup.md#settings).
2. Haz clic en el boton "**Registros**".
3. Selecciona "**Moderacion**".
4. Elige el canal en el que se enviaran los registros de sanciones o usa el boton "**Crear uno por mi**".

:::note
Tambien puedes elegir si RaidProtect registra las acciones realizadas por usuarios sin pasar por el bot.
:::
